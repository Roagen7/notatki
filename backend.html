  • dependencies: { "package-name": "version" } <---- zarządzanie wersją określonych modułów

Semantic Versioning (SemVer):

  • MAJOR: incompatible api changes <--- stuff that won't work with previous versions
  • MINOR: add functionality but that which will work with previous versions
  • PATCH: bug fixes
  • MAJOR.MINOR.PATCH np. 1.3.8  to major 1 minor 3 patch 8
  • ~version <--- update dependency to latest patch version np. ~1.3.6 allows to update to any 1.3.x version
  • ^version <--- update dependency to last minor version np. ^2.1.4 allows to update to any 2.x.x version
