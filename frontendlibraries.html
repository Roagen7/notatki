lifecycle of react component

  • Mounting
  • Updating
  • Unmounting

Mounting:

  • constructor(props) <---- initial values of state etc. called every time you make a component you should always start by calling super(props)
  •   static getDerivedStateFromProps(state) <--- returns object with changes to the state,    a place to change state properties based on props every time before render function
  • render() <--- putting JSX into DOM
  • componentDidMount() <---- here are methods that require the JSX being in the DOM already, called after the render method

Updating:

  • static getDerivedStateFromProps() <-- callled before update
  • shouldComponentUpdate() {return true/false} <---- check if the component should update its content called before update
  • render()
  • getSnapshotBeforeUpdate(prevProps,prevState) <--- wartości wszystkich propów i innych przed updatem called after update prevProps.propname
  • componentDidUpdate()    

Unmounting:

  • componentWillUnmount() <-- called when the object is removed from DOM

PODSUMOWANIE:

  • componentWillMount() <-- before first render
  • componentDidMount() <-- after mount <-- event handlery i change w state
  • componentWillReceiveProps(nextProps) <--- when receiving props
  • shouldComponentUpdate(nextProps,nextState) <-- before update <-- optimizing return true / false
  • componentWillUpdate() <-- before update
  • componentDidUpdate() <-- after component re-renders
  • componentWillUnmount()

REDUX STORE:

  • single state object managing state of entire web app
  • every time any piece of app wants to update it has to go through redux store
  • store = Redux.createStore(reducer)

metody:

  • Redux.createStore(reducer,middleware)
  • store.getState()
  • store.dispatch() <--- dispatching actions to store ->>> store.dispatch({type: 'LOGIN'})
  • store.subscribe()<--- add store listener ---> whenever an action is dispatched it is called ->>> store.subscribe(function)
  • Redux.combineReducers({id1: reducerFunctionName, id2: reducerFunctionName})
  • Redux.applyMiddleware(ReduxThunk.default)

action:

  • action - object with a 'type' property
  • np. {type: 'LOGIN'}
  • action creator <--- sending action to Redux store, function returning an action object
  • np. actionCreator = () => actionObject
  • middleware action ->> dać dispatch dopiero gdy otrzyma się dane

REDUCER:

  • reducer <--- function responding on dispatched actions
  • reducer(state = defaultState(), action){return modifiedAction}
  • state is read only
  • obsługiwanie wielu akcji: switch(action.type)
  • add different reducers to manage different parts of you webapp states and then combine them with rootReducer = combineReducers({key1: reducer1})
  • key1 contains all of the state associated with that part of the web app

keeping immutability:

  • [...spread]

  • Object.assign({},state) <--- tworzenie kopii statea
